# 💇 美容室予約システム（Backend担当）
本システムは、美容室の予約管理を中心に、
サロン情報・会員・認証機能を一元的に管理できるWebシステムである。


---

## 📌 概要（Overview）

- 美容室の予約業務を想定した Webアプリケーション
- Client・Designer・Managerで **役割別の機能分離**
- 実運用を意識した **予約重複チェック・時間計算ロジック** を実装

---

## 🏗️ 技術スタック

| 分類 | 使用技術 | 補足 |
|---|---|---|
| Backend | PHP 8.x | 予約ロジック・API 実装 |
| Web Server | Nginx | リバースプロキシ |
| Database | MySQL 8.x | 予約・ユーザー・サービス管理 |
| Infra / Dev | Docker Compose | Nginx / PHP-FPM / MySQL をコンテナ管理 |
| Router | AltoRouter | 軽量ルーティング |
| その他 | Postman | API 動作確認 |
|  | GitHub | ソースコード管理 |



---

## 👤 ユーザー区分（Role）

- **Client**  
  - メニュー・デザイナー・日時を選択して予約作成
  - 自分の予約履歴を確認
  - 予約を変更
  

- **Designer**  
  - 自分に紐づく予約の確認
  - 予約の状態を変更


- **Manager**  
  - デザイナーの休日を管理
  - サロンの運営に関する情報を管理（店舗情報、施術情報、ヘアスタイルカタログ、お知らせ）



---

## 🧩 主な機能
### 予約（コア）
- 複数サービス選択（例：カット + カラー）
- 予約重複チェック（同一時間帯・デザイナー）
- デザイナーの休日を考慮した予約制御
- サービス所要時間を考慮した終了時刻の自動計算
- 予約一覧表示（過去 / 未来）
- 役割別（Client / Designer）表示制御

### その他
- ログイン・会員登録（役割付与）
- サービス / デザイナー / サロン / ニュース / 一覧 / 取得

---


## 🧑‍💻 担当した役割（My Role）

- バックエンド全般を担当
- RESTful API の設計・実装
- 認証機能および予約機能の設計・実装
- 各種 CRUD ロジックの実装
- 業務ロジック（予約重複チェック・時間計算）の実装




## 🧠 工夫したポイント（Key Points）

- **DB設計を意識したテーブル分離**  
  （Reservation / Service / User など）
- サービスの所要時間を合算し、  
  **予約終了時刻を自動計算するロジック**を実装
- 実際の美容室運用を想定し、  
  「予約できないケース」を事前に判定
